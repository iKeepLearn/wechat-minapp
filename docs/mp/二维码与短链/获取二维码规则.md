# # 获取已设置的二维码规则

[调试诊断](https://developers.weixin.qq.com/console/devtools/debug?utm_source=api_tools)

> 接口应在服务器端调用，不可在前端（小程序、网页、APP等）直接调用，具体可参考接口调用指南。

接口英文名：qrcodeJumpGet

该接口用于获取已设置的二维码规则

## # 1. 调用方式

### # HTTPS 调用

### # 云调用

- 本接口不支持云调用。

### # 第三方调用

- 本接口支持第三方平台代商家调用。
- 该接口所属的权限集 id 为：3、18
- 服务商获得其中之一权限集授权后，可通过使用 authorizer_access_token 代商家进行调用，具体可查看 第三方调用 说明文档。

## # 2. 请求参数

### # 查询参数 Query String Parameters

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- ||
| access_token | string | 是 | 接口调用凭证，可使用 access_token、authorizer_access_token |

### # 请求体 Request Payload

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- ||
| appid | string | 是 | 小程序的appid。获取“扫服务号二维码打开小程序”已设置的二维码规则才需要传这个参数。 |
| get_type | number | 是 | 默认值为0。 0：查询最近新增 10000 条（数量大建议用1或者2）；1：prefix查询；2：分页查询，按新增顺序返回。获取“扫服务号二维码打开小程序”已设置的二维码规则才需要传这个参数。 |
| prefix_list | array | 否 | prefix查询，get_type=1 必传，最多传 200 个前缀。获取“扫服务号二维码打开小程序”已设置的二维码规则才需要传这个参数。 |
| page_num | number | 否 | 页码，get_type=2 必传，从 1 开始。获取“扫服务号二维码打开小程序”已设置的二维码规则才需要传这个参数。 |
| page_size | number | 否 | 每页数量，get_type=2 必传，最大为 200。获取“扫服务号二维码打开小程序”已设置的二维码规则才需要传这个参数。 |

## # 3. 返回参数

### # 返回体 Response Payload

| 参数名 | 类型 | 说明 |
| --- | --- | --- ||
| errcode | number | 错误码 |
| errmsg | string | 错误信息 |
| rule_list | objarray | 二维码规则详情列表 |
| qrcodejump_open | number | 是否已经打开二维码跳转链接设置 |
| list_size | number | 二维码规则数量 |
| qrcodejump_pub_quota | number | 本月还可发布的次数 |
| total_count | number | 二维码规则总数据量，用于分页查询 |

### # Res.rule_list(Array) Object Payload

二维码规则详情列表

| 参数名 | 类型 | 说明 |
| --- | --- | --- ||
| prefix | string | 二维码规则，说明，服务号二维码规则已过滤不展示。 |
| path | string | 小程序功能页面。 |
| state | number | 发布标志位，1 表示未发布，2 表示已发布。 |
| open_version | number | 测试范围。获取“扫普通二维码打开小程序”已设置的二维码规则才返回这个参数。1表示开发版（配置只对开发者生效）；2表示体验版（配置对管理员、体验者生效)；3表示正式版（配置对开发者、管理员和体验者生效）。特别说明，不管普通二维码规则是否发布，开发者/体验者/管理员在扫描 debug_url 所对应的二维码时，都会根据 open_version 来决定打开开发版、体验版或正式版 |
| debug_url | array | 测试链接（选填）可填写不多于 5 个用于测试的二维码完整链接，此链接必须符合已填写的二维码规则。获取“扫普通二维码打开小程序”已设置的二维码规则才返回这个参数。 |

## # 4. 注意事项

### # 调用说明

- 该接口用于获取已设置的二维码规则，适用于获取“扫普通二维码打开小程序”已设置的二维码规则，也适用于获取“扫服务号二维码打开小程序”已设置的二维码规则。
- 如果是获取“扫普通二维码打开小程序”已设置的二维码规则，则无需任何入参，使用小程序的[authorizer_access_token](https://developers.weixin.qq.com/doc/oplatform/openApi/ticket-token/api_getauthorizeraccesstoken)即可调用。
- 如果是获取“扫服务号二维码打开小程序”已设置的二维码规则，则需要传appid、get_type、prefix_list、page_num、page_size等参数，详情可查看下方文档说明。

### # 服务号调用说明

- 服务商要调用这个接口获取“扫服务号二维码打开小程序”已设置的二维码规则，必须要先获得服务号需将权限集 id 为3的[权限集](https://developers.weixin.qq.com/doc/oplatform/Third-party_Platforms/2.0/product/offical_account_authority.html)授权给服务商，否则会出现61007的报错。[排错指南](https://developers.weixin.qq.com/doc/oplatform/Third-party_Platforms/2.0/troubleshooting/third_party_authority_instructions_troubleshooting.html)
- 服务号要先关联小程序才可以调用该接口。如果服务号尚未关联小程序，可以通过关联小程序([linkMiniprogram](https://developers.weixin.qq.com/doc/oplatform/openApi/ams/percentage/api_getshareratio))的接口，或在“公众号管理后台—广告与服务—小程序管理”关联小程序。
